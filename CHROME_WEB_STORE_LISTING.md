## Permissions justification

**activeTab:** Inject UI into the current tab when you click the Pranan button or use a keyboard shortcut.

**storage:** Save your auth token and user preferences locally. No user data leaves your browser unless you explicitly invoke an action.

**sidePanel:** Display the Pranan side panel UI alongside your current page.

**tabs:** Detect when you've opened Gmail/Slack/LinkedIn so we know which content script to activate.

**cookies:** Read auth cookies from app.pranan.ai for sign-in handoff.

**alarms:** Periodic refresh of relationship context cache.

**webNavigation:** Detect URL changes (e.g., switching Gmail threads) to update the side panel.

**scripting:** Inject the "Draft with Pranan" button into compose toolbars.

**Host permissions** for mail.google.com / app.slack.com / linkedin.com / app.pranan.ai / *: required to read compose context and inject UI on the platforms we support. The wildcard host permission powers the Universal mode (selection-based rewrite/grammar on any site) — we never read DOM content unless you actively trigger an action.
